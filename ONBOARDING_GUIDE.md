# Guardian Override Onboarding Guide - v1.3_draft

Welcome to Guardian Override!  
This guide will help you get started as a contributor. It explains how to set up your environment, understand the project, and begin making meaningful contributions.

---

## Our mission

Guardian Override helps families set healthy screen boundaries with dignity and respect.  
We build tools that redirect attention gently, encourage reflection, and support trust — not surveillance or punishment.

## Setting up

1. **Clone the repository:**
   ```
   git clone https://github.com/litalien101/guardian-override.git
   cd guardian-override
   ```
2. **Install dependencies:**
   ```
   npm install
   ```
3. **Run the simulator**
   ```
   npm start
   ```
This will launch the prototype simulator where you can test rule behaviors and user flows.

---

## How to contribute

- **Report issues:** Use the Issues tab to share bugs, usability problems, or unclear documentation.

- **Suggest improvements:** Propose new features, rule schemas, or user flows that align with our mission.

- **Improve documentation:** Help make guides, onboarding, and explanations easier to understand.

- **Write code:** Contribute to modules like the simulator, schemas, extensions, or mobile scaffolds.

- **Review pull requests:** Give constructive feedback that is specific, kind, and focused on ideas.

---

## Contribution standards

- Use inclusive, welcoming language.

- Respect different viewpoints and experiences.

- Give feedback that is specific, actionable, and kind.

- Document assumptions and decisions clearly.

- Follow existing file structures and naming conventions.

- Keep user experience simple and accessible.

---

## Workflow

**1. Fork the repository** and create a branch from main.

**2. Make your changes** (code, docs, or tests).

**3. Test locally** to ensure everything works.

**4. Commit with clear messages.**

**5. Open a pull request** with a description of your changes.

---

## Support and questions

- Join the Discussions tab to ask questions or share ideas.

- Reach out to maintainers through the designated contact channel if you need help.

- **Remember:** no question is too small. We value clarity and collaboration.
